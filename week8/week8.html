Week 8

Overstap naar Tailwind CSS

Deze week was er een grote verandering: ik moest leren werken met Tailwind CSS. Dit betekende dat ik al mijn bestaande werk opnieuw moest bekijken en aanpassen aangezien ik dit niet eerder had gebruikt.

Moeilijkheden bij de overstap

In de vorige weken had ik alles netjes opgezet met aparte HTML-bestanden en eigen CSS-bestanden. Ik had mijn kleuren en styling allemaal in mijn eigen CSS geschreven, wat werkte maar best veel tijd kostte. Ik had eerder al ervaring met css en html en ik had het altijd al gedaan op die manier.

Toen kwam de opdracht: ik moest Tailwind CSS gaan gebruiken. In het begin dacht ik: waarom? Mijn eigen CSS werkt toch prima? Maar al snel begreep ik dat Tailwind een heel andere manier van werken is. In plaats van aparte CSS-bestanden schrijf je de styling direct in je HTML met utility classes. Dus in plaats van class="mijn-button" met aparte CSS, schrijf je nu iets als class="bg-blue-500 text-white px-4 py-2 rounded".

Dit betekende dat ik alles moest gaan veranderen. Al mijn bestaande pagina's met hun eigen CSS moesten worden omgebouwd naar Tailwind. Dat heeft best lang geduurd, want ik moest niet alleen Tailwind leren, maar ook al mijn oude werk opnieuw schrijven. Maar met behulp van AI ging het toch sneller dan als ik het alles helemaal opnieuw zelf had moeten uitschrijven.

Reflectie

Achteraf gezien was de overstap naar Tailwind best vervelend, vooral omdat ik al zoveel tijd had gestoken in mijn eigen CSS. Het voelde een beetje alsof ik opnieuw moest beginnen. Maar nu ik het een tijdje gebruik, zie ik wel de voordelen.

Met Tailwind kun je veel sneller prototypen en aanpassingen maken. Je hoeft niet steeds nieuwe class namen te verzinnen of te zoeken in je CSS-bestand waar iets staat. Je schrijft gewoon direct wat je wilt in de HTML. Het voelt efficiënter, ook al ziet je HTML er nu wat drukker uit met al die classes.

Ik heb ook gemerkt dat Tailwind je helpt om consistenter te zijn. Omdat je werkt met een vaste set aan utility classes (zoals px-4, px-8, enz.), gebruik je automatisch dezelfde spacing en kleuren door het hele project heen. Bij eigen CSS is het makkelijker om per ongeluk overal net iets andere waardes te gebruiken.

Conclusie

De overstap naar Tailwind CSS was best een uitdaging. Alles wat ik in aparte HTML's en CSS-bestanden had gemaakt, moest ik omzetten naar Tailwind's utility classes. Het kostte veel tijd en moeite om alles te herschrijven en de nieuwe manier van werken onder de knie te krijgen. Maar nu ik het eenmaal begrijp, merk ik dat het werken met Tailwind sneller en consistenter is. Het was een waardevolle les in flexibiliteit en het leren van nieuwe tools.
